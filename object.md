# ‚öñÔ∏è **Comparison: Native `Object` vs Lodash (`_.`)**

| **Feature / Use Case**               | **Native JS (`Object`)**                              | **Lodash (`_`)**                                                              | **Key Differences / Notes**                                                            |
| ------------------------------------ | ----------------------------------------------------- | ----------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| üîπ **Get Keys (Shallow)**            | `Object.keys(obj)`                                    | `_.keys(obj)`                                                                 | Both return **enumerable** own property names only. No difference for shallow objects. |
| üîπ **Get Keys (Deep)**               | ‚ùå Manual recursion required                           | ‚úÖ `_.keysIn()` (includes inherited) or `_.flatMapDeep()` for nested traversal | Lodash offers deeper and easier recursion options (`_.flatMapDeep`, `_.mapKeys`).      |
| üîπ **Get Values**                    | `Object.values(obj)`                                  | `_.values(obj)`                                                               | Both return enumerable values (only first level).                                      |
| üîπ **Get Entries (key-value pairs)** | `Object.entries(obj)`                                 | `_.toPairs(obj)`                                                              | Same output format. Lodash supports array-like and objects interchangeably.            |
| üîπ **Convert back to object**        | `Object.fromEntries(pairs)`                           | `_.fromPairs(pairs)`                                                          | Equivalent functionality. Lodash is slightly more flexible with arrays.                |
| üîπ **Deep Keys**                     | ‚ùå Requires recursion (manual)                         | ‚úÖ `_.keysIn()` + `_.forOwn` / `_.forIn` / custom `_.flatMapDeep`              | Lodash simplifies recursive key extraction.                                            |
| üîπ **Own vs Inherited Keys**         | `Object.keys()` = own only                            | `_.keysIn()` = own + inherited                                                | Lodash adds the inherited keys (like `for...in`).                                      |
| üîπ **Has Property**                  | `Object.hasOwn(obj, key)` / `obj.hasOwnProperty(key)` | `_.has(obj, path)`                                                            | Lodash supports **deep paths** like `'user.info.name'`, native does not.               |
| üîπ **Get Property Value**            | `obj[prop]`                                           | `_.get(obj, 'path.to.prop', defaultValue)`                                    | Lodash safely handles **undefined/null paths** and provides default fallback.          |
| üîπ **Set Property Value**            | `obj.prop = value`                                    | `_.set(obj, 'a.b.c', value)`                                                  | Lodash supports **deep assignment** ‚Äî auto-creates nested structure.                   |
| üîπ **Check Equality**                | `Object.is(a, b)` / `a === b`                         | `_.isEqual(a, b)`                                                             | Lodash does **deep comparison** for objects, arrays, nested structures.                |
| üîπ **Clone Object**                  | `structuredClone(obj)` / `{...obj}`                   | `_.clone()` / `_.cloneDeep()`                                                 | Lodash provides **deep cloning**, which native JS lacks (except `structuredClone`).    |
| üîπ **Merge Objects**                 | `Object.assign(target, src)`                          | `_.merge(target, src)`                                                        | Lodash performs **deep merge**; native does only shallow copy.                         |
| üîπ **Pick Properties**               | `{ key: obj.key }` (manual)                           | `_.pick(obj, ['key1', 'key2'])`                                               | Lodash simplifies selective key extraction.                                            |
| üîπ **Omit Properties**               | Manual destructuring                                  | `_.omit(obj, ['key'])`                                                        | Lodash makes removing specific keys easy.                                              |
| üîπ **Map Over Object**               | `Object.entries(obj).map(...)`                        | `_.mapValues(obj, fn)` / `_.mapKeys(obj, fn)`                                 | Lodash directly supports object mapping.                                               |
| üîπ **Flatten Keys (Deep)**           | Custom recursive function                             | `_.flatMapDeep(obj)`                                                          | Lodash provides built-in recursion utilities.                                          |

## üß† **Example: `Object.keys()` vs `_.keys()` vs `_.keysIn()`**

```js
const data = Object.create({ inherited: true });
data.name = "Alice";
data.info = { age: 30, city: "Tokyo" };

// Native
console.log(Object.keys(data));   // ['name', 'info']

// Lodash _.keys() - same as Object.keys
console.log(_.keys(data));        // ['name', 'info']

// Lodash _.keysIn() - includes inherited
console.log(_.keysIn(data));      // ['name', 'info', 'inherited']
```

## üß© **Deep Path Handling Example**

### ‚úÖ Lodash

### ‚ùå Native JS

```js

// Manual check required:

```

## üß© **Deep Keys Example**

### Native (Manual Recursive)

```js
function deepKeys(obj, prefix = '') {
  return Object.keys(obj).flatMap(k =>
    typeof obj[k] === 'object' && obj[k] !== null
      ? deepKeys(obj[k], `${prefix}${k}.`)
      : `${prefix}${k}`
  );
}

const data = { user: { name: 'Bob', address: { city: 'Tokyo' } }, active: true };
console.log(deepKeys(data));
// ['user.name', 'user.address.city', 'active']
```

### Lodash (Simpler)

```js
function lodashDeepKeys(obj, prefix = '') {
  return _.flatMap(_.keys(obj), k =>
    _.isObject(obj[k]) ? lodashDeepKeys(obj[k], `${prefix}${k}.`) : `${prefix}${k}`
  );
}

console.log(lodashDeepKeys(data));
// ['user.name', 'user.address.city', 'active']
```

## ‚ö° **Performance & Behavior Summary**

| Deep operations           | Requires recursion     | Easier with built-ins             |
| Handles nested paths      | ‚ùå No                   | ‚úÖ Yes (`_.get`, `_.set`, `_.has`) |
| Safe for `undefined/null` | ‚ùå No                   | ‚úÖ Yes                             |
| Supports default values   | ‚ùå No                   | ‚úÖ Yes                             |
| Works with arrays too     | Partially              | ‚úÖ Fully                           |
| Includes inherited props  | ‚ùå Only via manual      | ‚úÖ `_.keysIn()`                    |
| Library dependency        | None                   | Requires Lodash import            |

## üß© **When to Use Which**

| Use Case                                         | Recommendation                             |
| ------------------------------------------------ | ------------------------------------------ |
| Simple, shallow key/value extraction             | ‚úÖ **Native `Object` methods**              |
| Working with nested structures                   | ‚úÖ **Lodash (`_.get`, `_.set`, `_.merge`)** |
| Defensive programming (avoid `undefined` errors) | ‚úÖ **Lodash**                               |
| High-performance operations (tight loops)        | ‚úÖ **Native**                               |
| Readability & maintainability                    | ‚úÖ **Lodash**                               |

---

### TL;DR Summary

> * Use **`Object.keys()`** for **simple objects**.
> * Use **`_.get()`**, **`_.set()`**, or **`_.merge()`** for **nested or unpredictable data**.
> * Lodash simplifies deep work; native JS is faster for shallow operations.
